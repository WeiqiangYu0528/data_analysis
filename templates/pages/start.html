Application Structure

Web Framework

The web framework of the temporal graph management system is Django. Django is a high-level Python Web framework that encourages rapid development and clean, pragmatic design. Besides, Django also supports SQLites3 as the database by default.

Website Introduction

In this system, there are multiple pages to display the dynamic temporal network information. On the index page, people could see the general information of the datasets, like network types and network statistics data. Besides, the index page also contains the heat map and dataset space, which indicate the similarities between the datasets. What’s more, users could have access to the original data files by downloading them from the website.

On the data presentation page, users could see the details of each specific dataset. Furthermore, this page also includes several visualization components, such as force-directed graph, so that users are able to explore and understand the properties of the graph easily.

Deployment

The project is deployed on Linux by utilizing Apache2 and Virtualenv

File Structure

All the datasets are from SNAP. Stanford Network Analysis Platform (SNAP) is a general purpose network analysis and graph mining library. Visualization

Our system contains multiple visualization components. Hence, users could interact with the system freely, have an intuitive sense of network architecture and explore the properties of community structure

Data Distribution Map

Data distribution map allows users to observe the distribution(edges) of data in different time span.

In each specific dataset, users could choose different time intervals and time granularities, including month, quarter and year. Then the system will generate the corresponding data distribution map

Force-directed Graph

A force-directed diagram often exposes the inherent symmetric and clustered structure of a graph and shows a well-balanced distribution of nodes with few edge crossings.

Users can choose local or global community detection method, then the website will display the calculated communities. With different communities, I adopted different color points to represent them in the graph, which is very convenient for people to recognize.

Heatmap

The similarity matrix can be visualized as a heat map. The heatmap are sorted based on the distances from a given dataset.

User should first choose the similarity matrix, including Bhattacharyya coeﬃcient, Jensen–Shannon divergence and Vertex count. Furthermore, users are also able to combine similarity measures. For example, we can compare two graphs based on their degree distributions but also take under account their graph size. Afterwards, users should choose the time interval they want, and then the system will create the heat map accordingly

Dataset Space

The dataset space will be generated along with the heat map

Each dataset is projected to a point in a low-dimensional metric space that preserves the dataset similarities in the form of distances between the respective points

Graph Operator Prediction

Predict graph operator output over multiple graphs based on the calculated similarity matrix

Users should first select the graph operator they want to predict. Next, they shall choose the prediction ratio of samples. The system will calculate the actual value of graph operator for these samples, then estimate graph operators for the remaining graphs by running a weighted version of the k-Nearest-Neighbors (kNN) algorithm.

Community Structure

Community Detection

Community detection aims to identify highly connected groups of individuals or objects inside networks, these groups are called communities.

Global Community Detection

  • Focus on the concepts of subgroup cohesiveness and mutuality
    • k-cliques
    • k-cores
  • The internal and external vertex and subgraph degrees can also be used to deﬁne community.

Local Community Detection

  • Seen as a property of the whole network
  • One global community structure is based on the widely used concept of modularity
  • Another wide class of global community deﬁnitions relies on some similarity measure between network vertices

Louvain Algorithm

Louvain algorithm is a method to extract communities from large networks. It generally has two steps:

Step One

At the beginning, each node is regarded as a community, and the weight of edges in the community is 0. For each node, the algorithm traverses all the neighbor nodes and measures the modularity gain by adding the node to their communities. It then places the node into the community that resulted in the greatest modularity increase. This process is repeated until the communities do not change.

Step Two

In the second phase of the algorithm, it first groups all of the nodes in the same community and builds a new network. Next, it calculates the weight of the edges between these newly generated communities and between all nodes within the communities. Then it goes to the step one for the next round.

OLCPM

Online Label propagation CPM is a local community detection method.

Step One

It first implements OCPM, Online Clique Percolation Method, which takes two inputs: networks modification events and the clique size K. It returns all maximal cliques of size not less than k.

Step Two

The second step is label propagation, which sets out on the output communities of OCPM to discover the peripheral nodes. Each community spreads to neighboring peripheral nodes a label containing its identity and a weight representing the distance between this node and the community. When all labels have been shared, nodes belong to the communities with which they have the shortest distance. If the distances between the node and communities are the same, then the node belong to all of the communities.

Community Search

Community Search aims to provide efficient solutions for searching high-quality communities from large networks in real-time. Specifically, given a vertex q of a graph G, it aims to find a community, which contains q and satisfies the properties of connectivity and cohesiveness.

K-Core-Based Community Search

Community search with threshold constraint(CST):

For a given graph G(V, E), a query vertex q ∈ V , and a constant k, ﬁnd H ⊆ V such that

  1. q ∈ H
  2. G[H] is a connected graph
  3. min{ degG[H](v) | v∈V(H) } ≥ k

Step One

Determine whether the graph G contains the solution of the CST (k) of the vertex q

Step Two

Explore from the vicinity of the query vertex, and generate a candidate set C, which may contain a solution to the problem

Step Three

If the previous method fails to find a solution, a global search is performed on the k-cores of G[C] to find the solution.

Upload

Users are capble of uploading their own dataset files in our system.

  • First, users should fill in the required information for their dataset. For unknown information, just leave it blank
  • Secondly, users should select their dataset file
  • Then, our system will process the file you submitted.

Dataset File Format

The dataset file should at least contain three columns: SRC, DST and UNIXTS

  • SRC: id of the source node
  • TGT: id of the target node
  • UNIXTS: Unix timestamp

You can only upload a dataset file at a time

It may take a while to process the data file you uploaded. If the dataset file is uploaded successfully, you can go to extra datasets, and find the dataset you just uploaded.
